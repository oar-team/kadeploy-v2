kaconsole

NAME
	kaconsole - opens a console on a remote node

SYNOPSIS
	kaconsole -m|--machine hostname

DESCRIPTION
	kaconsole opens a console on a remote node. This tool is designed to help the user that would try to find out the reason of a deployment failure. It needs the appropriate command to be defined in the configuration file in order to be able to open a remote console on the given node ; see configuration file section for further details.

OPTIONS
	-m, --machine hostname
		specifies a host.

CONFIGURATION FILE
	/etc/kadeploy/deploy_cmd.conf
		Can contain four entry types per node whose only one is relative to the kaconsole command. This entry describes the command to open a remote console on the specified node of the cluster. Here is the description of the expected syntax :
		        host_name console command

		Example for idpot1 of idpot cluster :
			idpot1 console ssh -t localuser@ldap-idpot kermit -l /dev/ttyUSB0 -b 38400 -c

		In this example, kaconsole -m idpot1 means open a serial connection to idpot1 via kermit from ldap-idpot

EXAMPLE
	kaconsole -m node	# opens a console on node
